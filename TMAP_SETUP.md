# TMAP API 설정 가이드

이 프로젝트는 SK텔레콤의 TMAP을 사용하여 맛집 위치를 표시합니다.

## 1. TMAP API 키 발급

### 1.1 TMAP 개발자 사이트 회원가입
1. [TMAP API 사이트](https://openapi.sk.com/)에 접속합니다.
2. 우측 상단의 **"회원가입"** 버튼을 클릭합니다.
3. 약관에 동의하고 회원가입을 완료합니다.

### 1.2 앱 등록 및 API 키 발급
1. 로그인 후 **"마이페이지"** 또는 **"프로젝트 관리"**로 이동합니다.
2. **"앱 등록"** 또는 **"새 프로젝트 만들기"** 버튼을 클릭합니다.
3. 다음 정보를 입력합니다:
   - **프로젝트명**: matzip-chatgi (또는 원하는 이름)
   - **서비스 설명**: 맛집 추천 서비스
   - **사용 API**: "지도" 또는 "TMap" 선택
4. 등록을 완료하면 **App Key**가 발급됩니다.
5. 발급된 App Key를 복사합니다.

## 2. 환경 변수 설정

프로젝트 루트의 `.env.local` 파일을 열고 다음 줄을 수정합니다:

```env
NEXT_PUBLIC_TMAP_API_KEY=여기에_발급받은_API_키_붙여넣기
```

**예시:**
```env
NEXT_PUBLIC_TMAP_API_KEY=l7xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## 3. 개발 서버 재시작

환경 변수를 수정한 후에는 반드시 개발 서버를 재시작해야 합니다:

```bash
# 기존 서버 중지 (Ctrl + C)
# 서버 재시작
npm run dev
```

## 4. 확인

브라우저에서 다음 페이지들을 열어 지도가 정상적으로 표시되는지 확인합니다:
- http://localhost:3000/ (메인 페이지)
- http://localhost:3000/map-simple (간단한 지도 테스트)
- http://localhost:3000/map-test (고급 지도 테스트)

## 5. 요금 정보

TMAP API는 무료로 제공되며, 일일 호출 횟수 제한이 있을 수 있습니다. 자세한 내용은 TMAP API 사이트에서 확인하세요.

## 문제 해결

### "TMAP API 키가 설정되지 않았습니다" 메시지가 나타날 때
- `.env.local` 파일에 `NEXT_PUBLIC_TMAP_API_KEY`가 제대로 설정되었는지 확인하세요.
- 개발 서버를 재시작했는지 확인하세요.

### 지도가 표시되지 않거나 오류가 발생할 때
- TMAP API 키가 올바른지 확인하세요.
- 브라우저 개발자 도구(F12)의 콘솔에서 오류 메시지를 확인하세요.
- TMAP API 사이트에서 해당 프로젝트의 사용량 제한을 확인하세요.

### CORS 오류가 발생할 때
- TMAP API 사이트의 프로젝트 설정에서 허용된 도메인을 확인하세요.
- `http://localhost:3000`과 배포된 도메인을 허용 목록에 추가하세요.
